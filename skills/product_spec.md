# Skill: Product Specification

## Description
Transform a feature idea or user request into a complete product specification with requirements, acceptance criteria, edge cases, and test scenarios.

## Trigger
- New feature request from CEO or customer
- CPO roadmap planning session
- Linear ticket tagged with [FEATURE]

## Input
- Feature idea or user story
- Target user persona
- Business context and priority
- Technical constraints (if known)

## Output
- Product spec document with:
  - User story (As a [persona], I want [capability], so that [benefit])
  - Must-have requirements
  - Should-have requirements
  - Nice-to-have requirements
  - Acceptance criteria (testable)
  - Edge cases and error scenarios
  - Security considerations
  - UX wireframe description (text-based)
  - Success metrics

## Token Cost
MEDIUM — 3K-6K tokens per spec.
